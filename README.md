# Super Code — платформа курса (вайб-кодинг)

Веб-платформа для продажи видео-курса: регистрация, личный кабинет, уроки с
tier-гейтингом, защищённые видео через **Bunny Stream**, ручная оплата (чек →
проверка админом), геймификация (XP/уровни/лидерборд/викторина), AI-помощник
на Gemini и полноценная **админка** с мини-CRM (задачи, финансы, чаты).

Стек: **Next.js 14 (App Router) · PostgreSQL · Prisma · Tailwind · Bunny Stream · Gemini API**.

---

## Цель

Собрать рабочую платформу для продажи курса по вайб-кодингу под конкретный
бизнес (Super Code Academy, Таджикистан): лендинг → бесплатный урок →
регистрация → оплата по чеку → полный доступ, плюс админка для куратора,
которая закрывает все рутинные задачи (проверка чеков, ответы ученикам,
управление контентом) без стороннего SaaS.

## Текущее состояние

Проект в рабочем состоянии, крутится локально (`npm run dev`), БД — Postgres
через пользователя `course_bot_user` (переиспользован из отдельного Telegram-
бота заказчика, база `supercode` отдельная). Все ключевые пользовательские
и админские сценарии реализованы и вручную проверены через реальные HTTP-
запросы с валидными сессиями (см. «Что пробовали» — из-за особенностей Next
Server Actions полноценных браузерных e2e-тестов нет, только curl/скрипты).

### Ученик

- Лендинг с hero, каруселью отзывов, блоком "Почему так", FAQ, футером —
  переключатель языка RU/ТҶ (черновой перевод, не проверен носителем)
- Регистрация (email + телефон, оба обязательны и уникальны), вход по
  email **или** телефону
- Личный кабинет (`/dashboard`) с боковым сайдбаром (десктоп) / плавающими
  верхней плашкой + нижним таб-баром (мобайл): прогресс, уровень, XP, ранг
  в рейтинге, топ-3 ученика, список модулей/уроков
- Страница урока: видео-плеер (Bunny, токенизированная ссылка с TTL),
  навигация "Назад/Далее" по всем урокам курса, программа курса сбоку
  (десктоп) / по кнопке (мобайл), отметка "пройдено" с начислением XP
- Оплата — пошаговый визард на отдельных страницах: тариф → способ оплаты →
  реквизиты + чек → подтверждение (не одна страница со сменой шагов, а
  реальные `/billing/checkout/*` роуты)
- **Геймификация**: XP за уроки и викторину, уровни (расширяемая
  прогрессия), лидерборд (топ-50 + своя позиция), викторина для новичков
  (8 вопросов по темам модулей 1-2, без Pro-контента)
- **Super AI** (`/ai`) — общий чат-помощник на Gemini, история сохраняется
  в БД, rate limit 20 сообщений/час на пользователя
- **Чат с куратором** (`/support`) — простой двусторонний чат ученик↔админ,
  доступен из профиля
- Колокольчик уведомлений (плавающий, на всех страницах кабинета) —
  поздравления при прохождении уроков/викторины
- Профиль: аватар, инлайн-редактирование имени, смена пароля, история
  платежей, блок быстрого доступа

### Админ

- Категоризированный сайдбар: Обзор, Задачи, Продажи и учёт (Платежи,
  Реквизиты), Финансы, Ученики, Обучение (Уроки, Отзывы, Викторина), Чаты
- Статистика, платежи (approve/reject с комментарием), уроки и модули,
  пользователи, реквизиты оплаты, отзывы (для карусели на лендинге)
- Задачи — простой тудушный список (открыта/в работе/готова)
- Финансы — выручка из одобренных платежей + расходы вручную, чистыми
- Чаты — список тредов по ученикам + сама переписка
- Викторина — вопросы с подсветкой правильного варианта, добавление/удаление

## Что пробовали и не сработало (важно для следующего раза)

- **`gemini-2.5-flash` недоступен новым API-ключам** — Google отдаёт 404
  "no longer available to new users", хотя модель числится в `/v1beta/models`.
  Решение — использовать алиас `gemini-flash-latest` (`src/lib/gemini.ts`),
  Google сам держит его указывающим на актуальную flash-модель.
- **Дублирующийся SVG `id` ломает иконку в Safari.** `NeuroIcon` рендерился
  дважды на одной странице (видимый мобильный хедер + скрытый через
  `display:none` десктопный сайдбар) с одинаковым `id="neuro-grad"` у
  `<linearGradient>`. Safari резолвит `url(#id)` в первый элемент с таким id
  в DOM — если это скрытый элемент, градиент не рисуется вообще. Первая
  гипотеза (конфликт `backdrop-filter` + `background-clip: text`) была
  **неверной** — реальная причина именно дубли id. Теперь `id` передаётся
  пропом, у каждого места использования свой.
- **Prisma Client кэшируется в dev.** Синглтон в `src/lib/db.ts` живёт в
  `globalThis`, и после `prisma migrate dev` уже запущенный `next dev` не
  подхватывает новый клиент — новые модели/поля падают с "not a function"
  или undefined, пока не перезапустишь dev-сервер целиком (`rm -rf .next`
  тоже не помогает без рестарта процесса).
- **`prisma migrate dev` требует `CREATEDB` у роли БД** (нужен для shadow-
  database). У `course_bot_user` изначально этого права не было — пришлось
  выдать `ALTER ROLE course_bot_user CREATEDB;`.
- **`overflow-x-hidden` на предке ломает `position: sticky` на iOS Safari.**
  Плавающий хедер лендинга переставал "прилипать" при скролле. Причина —
  `overflow-x: hidden` без явного `overflow-y` неявно создаёт скролл-
  контейнер, который в мобильном Safari конфликтует со sticky-потомками.
  Заменили на `overflow-x-clip` (не создаёт скролл-контекст).
- **Установка Node.js через `brew` зависала** на скачивании с `ghcr.io`
  (реестр Homebrew) — соединение было почти нулевым. Помог только повтор
  команды (со второго раза `ghcr.io` отдавал нормально).
- Полноценные браузерные e2e-тесты не настроены — React Server Actions
  используют внутреннее кодирование (`$ACTION_ID`), которое не
  воспроизвести обычным `curl`. Проверка велась через: (а) прямые HTTP-
  запросы с валидной JWT-сессией к серверным страницам, (б) отдельные
  Node-скрипты, копирующие бизнес-логику экшена 1:в:1 против реальной БД
  (начисление XP, подсчёт очков викторины, read-флаги чата), с последующей
  очисткой тестовых данных.

## Запуск локально (пошагово)

1. **Установите Node.js 18+** и PostgreSQL (или возьмите облачную базу: Neon,
   Supabase, Railway — там дают готовый `DATABASE_URL`).

2. **Установите зависимости** в папке проекта:
   ```bash
   npm install
   ```

3. **Настройте переменные окружения.** Скопируйте пример и заполните:
   ```bash
   cp .env.example .env
   ```
   Минимум для старта — `DATABASE_URL` и `AUTH_SECRET`. Bunny и Gemini можно
   подключить позже (без Bunny уроки работают с заглушкой вместо видео, без
   `GEMINI_API_KEY` не заработает раздел Super AI).

   Сгенерировать `AUTH_SECRET`:
   ```bash
   openssl rand -base64 48
   ```

4. **Создайте таблицы и демо-данные:**
   ```bash
   npm run db:migrate  # применяет миграции Prisma
   npm run db:seed      # заполняет демо-курсом, викториной, админом и учеником
   ```

5. **Запустите:**
   ```bash
   npm run dev
   ```
   Откройте http://localhost:3000

   ⚠️ После любого `prisma migrate dev` перезапускайте `npm run dev` целиком —
   см. «Что пробовали и не сработало» выше (Prisma Client кэшируется в dev).

## Подключение Bunny Stream (видео)

1. В bunny.net создайте **Stream → Video Library**.
2. В `.env` заполните:
   - `BUNNY_STREAM_LIBRARY_ID` — ID библиотеки
   - `BUNNY_STREAM_API_KEY` — ключ библиотеки (для загрузки видео из админки)
   - `BUNNY_STREAM_TOKEN_KEY` — **Security → Token Authentication Key**
   - `BUNNY_STREAM_CDN_HOSTNAME` / `BUNNY_STREAM_EMBED_HOSTNAME`
3. В библиотеке включите **Token Authentication**.
4. Загрузите видео (в панели Bunny или кнопкой «Создать видео в Bunny» в админке)
   и впишите его `guid` в поле урока «Bunny Video ID».

Ссылка на плеер подписывается на сервере (`src/lib/bunny.ts`) с TTL 15 минут —
просто скопировать URL в другую вкладку надолго не получится.

## Подключение Super AI (Gemini)

1. Получите ключ на [aistudio.google.com/apikey](https://aistudio.google.com/apikey).
2. В `.env` впишите `GEMINI_API_KEY`.
3. Модель зафиксирована как `gemini-flash-latest` в `src/lib/gemini.ts` —
   версии вида `gemini-2.5-flash` могут быть недоступны новым ключам (см.
   «Что пробовали и не сработало»).

## Деплой на Vercel

Проект готов к Vercel (лендинг уже там). Важно:
- Задайте те же переменные окружения в настройках проекта Vercel.
- Файловая система Vercel **только для чтения**, поэтому загрузку скриншотов
  чеков и видео отзывов (`src/lib/storage.ts`) для продакшена нужно перевести
  на объектное хранилище (Bunny Storage / S3 / Supabase Storage). Локально и
  на обычном VPS запись в `public/uploads` работает сразу.

## Структура

```
prisma/schema.prisma      — схема БД: users, courses/modules/lessons,
                            lesson_progress, payments, payment_methods,
                            reviews, tasks, expenses, chat_messages,
                            ai_chat_messages, notifications,
                            quizzes/quiz_questions/quiz_attempts
prisma/seed.ts            — демо-данные (курс, викторина, админ, ученик)

src/lib/                  — db, auth (JWT), access (тарифы), bunny, gemini,
                            gamification (XP/уровни), ordinal, i18n/landing,
                            pricing, storage, validation
src/middleware.ts         — защита приватных и админских разделов

src/app/(auth)/           — /login, /register
src/app/(dashboard)/      — общий сайдбар/мобильная навигация:
                            /dashboard, /billing(+/checkout/*), /profile,
                            /leaderboard, /lessons/[id], /support, /ai,
                            /quiz(+/play, /result)
src/app/admin/            — админка: /admin, /payments, /lessons, /users,
                            /reviews, /methods, /tasks, /finance, /chats,
                            /quiz
src/app/api/ai/chat/      — route handler для Super AI (fetch, не server action —
                            нужен для индикатора "печатает…" на клиенте)
src/app/actions/          — server actions (auth, admin, payments, progress,
                            profile, chat, notifications, quiz)

src/components/           — переиспользуемые UI-компоненты (DashboardSidebar,
                            AdminSidebar, LandingHeader/Hero/... , AiChatUI,
                            NotificationBell, LessonProgram, и т.д.)
```

Код прокомментирован по-русски — мелкие правки можно вносить через ИИ-инструменты.
